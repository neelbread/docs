## Welcome to Bread

The Bread Python SDK provides convenient access to "Git for Weight Space," a version control system that lets you make weight updates to AI models. The library includes type definitions for all request parameters and response fields, and offers both synchronous and asynchronous clients.

## Features

<AccordionGroup>
  <Accordion title="Type-Safe & Modern" icon="shield-check">
    Full type definitions for all request params and response fields using Pydantic models and TypedDicts
  </Accordion>

  <Accordion title="Async & Sync Support" icon="bolt">
    Both synchronous and asynchronous clients powered by httpx, with optional aiohttp support for improved concurrency
  </Accordion>

  <Accordion title="Comprehensive SDK" icon="cube">
    Complete coverage of Bread API resources: Repositories, Prompts, Targets (Stim & Rollout), Bakes, and Health checks
  </Accordion>

  <Accordion title="Production Ready" icon="check-double">
    Built-in retry logic, timeout configuration, error handling, and streaming response support
  </Accordion>
</AccordionGroup>

## Quick Example

```python
import os
from aibread import Bread

client = Bread(
    api_key=os.environ.get("BREAD_API_KEY")
)

# Create a repository
repo = client.repo.set(repo_name="my_repo")

# Create prompts
client.prompts.set(
    prompt_name="system_prompt",
    repo_name="my_repo",
    messages=[{"role": "system", "content": "You are a coding assistant"}]
)

# Configure and run a target
target = client.targets.set(
    target_name="coding_target",
    repo_name="my_repo",
    template="default",
    overrides={
        "model_name": "Qwen/Qwen3-32B",
        "generators": [{"type": "hardcoded", "numq": 1, "questions": ["Write a function to reverse a string"]}]
    }
)

# Generate stimuli
client.targets.stim.run(target_name="coding_target", repo_name="my_repo")
```

## Requirements

- **Python**: 3.8 or higher
- **Dependencies**: httpx (included), optional aiohttp for async performance
- **API Key**: Required for authentication
